## 1. Brand Foundation

### Golden Circle (Simon Sinek)

```
         ┌─────────┐
         │   WHY   │
         └────┬────┘
          ┌───┴───┐
          │  HOW  │
          └───┬───┘
       ┌──────┴──────┐
       │    WHAT     │
       └─────────────┘
```

**WHY — O Porquê**
> Acreditamos que cada negócio merece software que foi construído para ele — não repropositado para ele. Acreditamos que tecnologia deve servir o negócio, não o contrário.

**HOW — Como Agimos**
- Briefing profundo antes de qualquer linha de código
- Construção de raiz — nunca templates adaptados
- Arquitectura limpa, documentada, escalável
- Desenvolvimento iterativo com updates regulares
- Parceria técnica contínua após entrega
- Código que o cliente consegue manter e evoluir

**WHAT — O Que Entregamos**
- Web Apps à medida
- Landing Pages de conversão
- Automações e pipelines
- Arquitectura de software modular

**Comunicação Inside-Out:**
> *"Acreditamos que nenhum negócio deveria estar preso a software que não foi feito para ele. Por isso construímos de raiz, com arquitectura limpa e parceria real. O que fazemos: web apps, landing pages e automações à medida para startups e PMEs."*

## 2. Posicionamento

### Análise Competitiva

| Tipo | Posição | Problema |
|------|---------|---------|
| Agências genéricas | Templates rápidos, portfolio grande | Entregam o mesmo para todos |
| Freelancers | Preço baixo, entrega rápida | Sem estrutura, desaparecem |
| Consultoras grandes | Metodologia pesada, enterprise | Caras demais para startups |
| No-code platforms | Veloz, sem código | Não escala, não é teu |
| Dev shops offshore | Price-first | Qualidade variável, sem parceria |

### Posição a Ocupar

NodeGrid ocupa o espaço entre "freelancer" e "agência enterprise" — arquitectura de nível enterprise para escala e orçamento startup.

**Linha de Posicionamento:**
> Para startups e PMEs que foram traídas por templates e agências que desaparecem, a NodeGrid é o parceiro técnico que constrói de raiz e fica — porque o seu sucesso é o nosso sucesso.

**Posição de Mercado:**
> "O parceiro técnico para startups e PMEs que cresceram além dos sistemas genéricos."

**A Palavra a Possuir:**
```
ARQUITECTURA
```

## 3. Arquétipo de Marca

### Arquétipo Principal: O CRIADOR

| Dimensão | Conteúdo |
|----------|---------|
| Essência | Criar algo que dure e que tenha valor real |
| Desejo | Dar forma a uma visão com precisão técnica |
| Medo | Mediocridade, trabalho descartável, código sujo |
| Estratégia | Construir com intenção, não de forma apressada |
| Tom | Confiante, preciso, artesanal |

### Arquétipo Secundário: O REBELDE

| Dimensão | Conteúdo |
|----------|---------|
| Essência | Desafiar o status quo das agências genéricas |
| Desejo | Mudar a forma como software é feito e vendido |
| Estratégia | Posicionar contra o que todos os outros fazem |
| Tom | Directo, sem filtros, anti-clichés |

### Combinação Criador + Rebelde
> NodeGrid é o artesão que se recusa a trabalhar com moldes. Constrói porque acredita que cada peça merece ser única. E diz-o sem rodeios.

### Implicações para Voz de Marca

| Fazer | Não Fazer |
|-------|----------|
| Técnico mas humano | Jargão corporativo |
| Directo e sem exageros | "Solução disruptiva" |
| Mostrar o trabalho | Listar tecnologias sem contexto |
| Orgulho artesanal | Promessas vagas |
| Anti-bullshit | Buzzwords |

## 4. StoryBrand Framework

### BrandScript Completo

**O Herói:**
Fundador de startup ou PME com um negócio real para construir — sério, exigente, farto de genérico.

**O Problema:**
- **Externo:** Software genérico que não serve o seu fluxo de trabalho
- **Interno:** Frustração de pagar por soluções que não encaixam; medo de ser abandonado após a entrega
- **Filosófico:** O teu negócio merece tecnologia feita para ele

**O Guia (NodeGrid):**
- Empatia: *"Sabemos como é ficar preso num sistema que não é teu"*
- Autoridade: Next.js · Supabase · Arquitectura limpa · MVPs lançados

**O Plano:**
1. Conta-nos o teu projeto (briefing sem compromisso)
2. Proposta clara com timeline e âmbito definidos
3. Desenvolvimento iterativo com updates regulares
4. Entrega + parceria técnica contínua

**Calls to Action:**
- Directo: *"Falar no WhatsApp"*
- Passivo: *"Ver como trabalhamos"*

**Em Risco se Não Agir:**
- Mais dinheiro gasto em templates que não escalam
- Mais código legado impossível de manter
- Mais agências que somem após a factura

**Sucesso:**
- Sistema construído exactamente para o seu negócio
- Arquitectura limpa que a equipa consegue manter
- Um parceiro técnico que atende quando precisas

### Headlines StoryBrand-Ready

**Headline principal:**
> "O teu negócio não é genérico. O teu software também não devia ser."

**Logline de marca:**
> "Construímos o software que o teu negócio merecia desde o início — de raiz, com arquitectura limpa, e ficamos quando precisas de nós."

## 6. Brand Differentiation — The Zag

### Onlyness Statement

> NodeGrid é o único software studio para startups e PMEs que aplica pensamento arquitectural de enterprise a projectos custom — e permanece como parceiro técnico após a entrega, não como mais um fornecedor que some.

### Stress-Test

| Questão | Resposta |
|---------|----------|
| É verdade? | Sim — validado pela postura e modelo de negócio |
| É sustentável? | Sim — modelo de parceria cria lock-in positivo |
| É difícil de copiar? | Cultura difícil, palavras fáceis |
| Comunica em 3 segundos? | Com a mensagem certa, sim |

### A Palavra Emocional da Marca

```
CRAFT
```

> ARQUITECTURA é o racional (o que fazem).
> CRAFT é o emocional (quem são).
> Craft implica: feito à mão, com cuidado, não replicável.
> Combina Criador + Rebelde sem precisar de explicar.

## 7. Identidade Visual

### Sistema de Cores

| Token | Valor | Uso |
|-------|-------|-----|
| `--color-bg` | `#070e07` | Background principal |
| `--color-surface` | `#0d140d` | Cards e superfícies |
| `--color-surface-2` | `#111811` | Superfícies secundárias |
| `--color-green` | `#72b872` | Accent principal, links, tags |
| `--color-green-dim` | `#2a5a2a` | Borders, estados hover |
| `--color-gold` | `#f0d474` | CTA exclusivo — não usar para outro fim |
| `--color-text-1` | `#dcdcdc` | Texto principal |
| `--color-text-2` | `#787878` | Texto secundário |

### Tipografia

| Font | Uso | Peso |
|------|-----|------|
| Inter | Corpo, headings, UI | 400, 500, 600, 700 |
| JetBrains Mono | Labels técnicos, tags, código, metadata | 400, 500 |

**Regra:** JetBrains Mono é a assinatura técnica da marca. Usar em section tags, eyebrows, badges, e qualquer elemento que sinalize expertise técnica.

### Auditoria Visual

| Critério | Score | Estado |
|----------|-------|--------|
| Distinção | 8/10 | Forest green é raro no mercado PT |
| Consistência | 7/10 | Forte na landing, sem extensão |
| Escalabilidade | 5/10 | PNG logo limita aplicações |
| Memorabilidade | 8/10 | Paleta + mono font = assinatura |
| Alinhamento | 9/10 | Visual alinha com arquétipo |
| **Global** | **7.4/10** | **Base forte, sistematização urgente** |

### Decisões Certas a Proteger

- Paleta dark forest green — rara, distinctiva, ownable
- JetBrains Mono — credibilidade técnica genuína
- Gold `#f0d474` como CTA único — contraste perfeito
- Ausência de blue/purple tech clichés
- Card-based UI com borders subtis

### Fixes por Prioridade

**Crítico (fazer já)**
1. Logo → converter para SVG com variantes light/dark/mono
2. Brand guidelines 1-pager (cores, fontes, logo usage, do/don't)
3. Favicon e Open Graph image consistentes

**Importante (próximo mês)**
4. Icon system — line icons, 1.5px stroke, Phosphor ou Lucide
5. Templates de posts sociais (LinkedIn + Instagram)
6. Email signature padronizada
7. Proposal/documento template

**Desejável (trimestre)**
8. Motion principles (micro-animations alinhadas com craft)
9. Padrão/textura de marca
10. Slide deck template para apresentações a clientes

## 9. Cultura de Marca

### Valores → Implicações Operacionais

| Valor Declarado | Implicação Cultural Obrigatória |
|----------------|--------------------------------|
| "Código limpo" | Code review em TODOS os projectos — sem excepções |
| "Arquitectura" | Brief técnico antes de qualquer proposta |
| "Sem surpresas" | Timelines honestos mesmo que incomodem |
| "Parceria contínua" | Check-in formal 30/60/90 dias pós-entrega |
| "Anti-genérico" | Recusar projectos que não permitem craft |

### Riscos de Desalinhamento

**Risco 1 — Volume vs. Craft**
Aceitar projectos a mais degrada a qualidade. Limite de capacidade é uma decisão de marca.

**Risco 2 — Entrega vs. Continuidade**
Entregar sem follow-up mata "parceria contínua". A promessa de ficarem tem de ser um processo, não uma intenção.

**Risco 3 — Linguagem Interna vs. Externa**
Usar buzzwords internamente que recusam externamente contradiz a marca. Anti-bullshit começa dentro.

### Plano de Alinhamento (4 Acções)

1. **Delivery Quality Checklist** — Checklist de entrega que é os valores em acção
2. **Protocolo de Continuidade** — Quem contacta, quando (30/60/90 dias), o que pergunta
3. **Critérios de Aceitação de Projecto** — Brief completo? Arquitectura possível? Cabe na capacidade?
4. **Celebrar Craft, não Velocidade** — O que se celebra define o que se repete

### Nota sobre nodegrid.dev

`.dev` é o TLD operado pela Google exclusivamente para tecnologia e desenvolvimento. HTTPS-only por definição (SSL obrigatório). Reforça o posicionamento técnico imediatamente — sem explicação necessária. É um asset de marca, não apenas um domínio.

### Acções Pendentes

1. **Trademark (60 dias)** — Pesquisa INPI (PT) + EUIPO (EU) Classe 42; se disponível, registar imediatamente
2. **Domínios adicionais** — Assegurar `nodegrid.pt` como redirect para `.dev`
3. **Handles sociais** — Reclamar `@nodegrid` em LinkedIn, GitHub, Instagram, X
