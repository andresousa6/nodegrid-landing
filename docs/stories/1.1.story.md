## Story

**Como** visitante da NodeGrid Landing Page,  
**Quero** aceder a uma página de portfolio com os projectos desenvolvidos pela equipa,  
**Para** perceber o tipo de trabalho produzido e ganhar confiança para contratar.

---

## Contexto

A NodeGrid LP (`workspaces/nodegrid-landing/index.html`) existe e está funcional. É HTML/CSS estático com o design system definido em `css/tokens.css` (fundo `#070e07`, verde `#72b872`, gold `#f0d474`).

Precisamos de:
1. Adicionar link "Portfolio" na nav da LP
2. Criar `portfolio.html` no mesmo directório com grid de projectos
3. Tirar screenshots automáticos com Playwright dos projectos listados
4. Integrar os screenshots nos cards do portfolio

### Projectos a incluir

| Projecto | Workspace | Stack | Porta dev |
|----------|-----------|-------|-----------|
| Video-TaskManager | `workspaces/Video-TaskManager` | Next.js 14, Supabase, TanStack Query | 3000 |
| Hidden Hype | `workspaces/hidden_hype` | Next.js 15, Supabase, next-pwa | 3001 |
| Hype-Gest (EventOS) | `workspaces/hype-gest/frontend` | React/Next.js | 3002 |
| Timetable Contracts | `workspaces/timetable-contracts` | Next.js, Supabase | 3003 |

## Acceptance Criteria

### AC1 — Nav da LP actualizada

- [ ] Link aparece também no footer da LP (secção de navegação)

### AC2 — Página `portfolio.html` criada
- [ ] Ficheiro existe em `workspaces/nodegrid-landing/portfolio.html`
- [ ] Usa os mesmos tokens CSS da LP (`css/tokens.css`)
- [ ] Nav idêntica à LP (com "Portfolio" em estado activo/highlight)
- [ ] Footer idêntico ao da LP
- [ ] Título de página: `Portfolio — NodeGrid`

### AC3 — Grid de projectos
- [ ] 4 cards de projectos exibidos em grid responsivo (2 colunas desktop, 1 coluna mobile)
- [ ] Cada card contém:
  - Imagem/screenshot do projecto (ou placeholder com cor de marca)
  - Nome do projecto
  - Badge de categoria (ex: "Gestão de Produção", "Eventos & Ticketing")
  - Descrição curta (máximo 2 linhas)
  - Lista de 3 features principais
  - Chips de tech stack (ex: Next.js, Supabase, TypeScript)
- [ ] Sem botões de "Ver Demo" (não há URLs live por agora)
- [ ] Cards têm hover state com elevação/brilho subtil

### AC4 — Conteúdo dos cards

**Video-TaskManager**
- Categoria: `Gestão de Produção`
- Descrição: Plataforma de gestão de tarefas para equipas de produção de vídeo com dashboard para directores, vista de marketer e fluxo de aprovação integrado.
- Features: Dashboard de directores com métricas em tempo real · Atribuição e tracking de tarefas por membro · Fluxo de aprovação de conteúdos
- Stack: Next.js 14, Supabase, TanStack Query, TypeScript

**Hidden Hype**
- Categoria: `Eventos & Ticketing`
- Descrição: App de gestão de eventos com check-in por QR code, autenticação via telemóvel (OTP) e painel de administração para controlo de acessos.
- Features: Check-in instantâneo via QR code · Autenticação por OTP (telemóvel) · Painel admin com controlo de acessos
- Stack: Next.js 15, Supabase, PWA, TypeScript

**Hype-Gest (EventOS)**
- Categoria: `Gestão de Eventos`
- Descrição: Plataforma completa para gestão do ciclo de vida de eventos — do planeamento à execução, com Gantt interactivo, gestão de riders e previsão de cashflow.
- Features: Gantt interactivo com arrastar e largar · Sistema de riders e requisitos técnicos · Previsão de orçamento e cashflow
- Stack: Next.js, Supabase, TypeScript, React

**Timetable Contracts**
- Categoria: `Contratos Digitais`
- Descrição: Gerador de contratos com assinatura digital para artistas e fornecedores, com templates personalizáveis e exportação PDF.
- Features: Geração automática de contratos PDF · Assinatura digital integrada · Templates personalizáveis por cliente
- Stack: Next.js, Supabase, TypeScript, pdf-lib

### AC5 — Screenshots via Playwright
- [ ] Script `scripts/take-screenshots.js` criado em `nodegrid-landing/`
- [ ] Script inicia cada dev server (se não estiver a correr), navega para a página principal e tira screenshot
- [ ] Screenshots guardados em `nodegrid-landing/assets/screenshots/` com nomes: `video-task-manager.png`, `hidden-hype.png`, `hype-gest.png`, `timetable-contracts.png`
- [ ] Screenshots têm dimensão 1200×800px (16:9 landscape)
- [ ] Se o dev server falhar ao iniciar, o script usa um placeholder gerado com a cor de marca do projecto
- [ ] Instrução de uso documentada no `README.md` da LP (criar se não existir)

### AC6 — Qualidade visual
- [ ] Design consistente com a LP (mesmo tipografia, espaçamentos, paleta)
- [ ] Página é responsiva (testada em mobile 375px e desktop 1280px)
- [ ] Sem consola de erros JavaScript
- [ ] Sem quebras de layout óbvias em Safari/Chrome

## Notas técnicas

- A LP não usa bundler — todo o CSS é vanilla, sem Tailwind ou frameworks. Manter o mesmo padrão no `portfolio.html`.
- O script de screenshots deve usar `playwright` (verificar se disponível globalmente: `npx playwright --version`). Se não disponível, usar `puppeteer` como fallback.
- Cada projecto tem a sua cor de marca para o placeholder:
  - Video-TaskManager: `#35998f` (teal)
  - Hidden Hype: `#C9A84C` (gold)
  - Hype-Gest: `#4a9a4a` (verde)
  - Timetable Contracts: `#B89A5E` (brass)
- Para iniciar os dev servers nos screenshots, usar portas distintas (3000–3003) e aguardar `localhost:PORT` responder antes de navegar.
