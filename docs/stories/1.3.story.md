# Story 1.3 — Páginas de Detalhe de Projecto NodeGrid

**Epic:** 1 — NodeGrid Website
**Story:** 1.3
**Status:** Ready

## Story

**Como** visitante que clicou num projecto no portfolio,
**Quero** ver uma página rica com demonstração, features, design e stack de cada projecto,
**Para** perceber em detalhe o que a NodeGrid construiu e ganhar confiança para contratar.

---

## Contexto & Referência

Site de referência: `https://design.aioxsquad.ai/linear`

**Estrutura da página de referência:**
- Breadcrumb + PREV/NEXT entre projectos
- Nome do projecto em tipografia MASSIVA (display)
- Tabs sticky: VISÃO GERAL | FUNCIONALIDADES | STACK | DESIGN
- `01 // APP_PREVIEW` — browser frame mockup com screenshot
- `02 // SOBRE` — narrativa problema → solução
- `03 // FUNCIONALIDADES` — 3 features com detalhe
- `04 // PALETTE` — color blocks do projecto (BACKGROUND, SURFACE, ACCENT, CTA, TEXT)
- `05 // STACK` — tecnologias com descrição

**Design system:** tokens NodeGrid (`css/tokens.css`) + cor de marca de cada projecto no hero.

## Conteúdo por Projecto

### [01] Video-TaskManager

**Ficheiro:** `projeto/video-taskmanager.html`
**Slug:** `video-taskmanager`
**Cor de marca:** `#35998f` (teal)
**PREV:** `timetable-contracts.html` | **NEXT:** `hidden-hype.html`

**Hero:**
- Nome: `VIDEO-TASKMANAGER`
- Categoria: `GESTÃO DE PRODUÇÃO`
- URL: `videotask.nodegrid.dev`

**02 // SOBRE:**
- Headline: *"Equipas de vídeo a gerir tarefas por WhatsApp. Havia uma forma melhor."*
- Problema: Equipas de produção de vídeo dependem de WhatsApp e sheets para gerir tarefas, aprovações e deadlines. O resultado é caos, versões perdidas e atrasos recorrentes.
- Solução: Dashboard centralizado com visões específicas por função (director, marketer, criador) e fluxo de aprovação estruturado com notificações em tempo real.
- Métricas: `STACK: 4 tecnologias` | `TIPO: Web App` | `AUTH: Supabase` | `UI: Radix + Sonner`

**03 // FUNCIONALIDADES:**
1. **Dashboard Director** — Visão global de todas as tarefas da equipa com métricas em tempo real, assignment por membro e gestão de deadlines num só ecrã.
2. **Vista Marketer** — Interface personalizada com tarefas atribuídas, upload de assets de campanha e histórico completo de aprovações.
3. **Fluxo de Aprovação** — Ciclo estruturado submissão → revisão → aprovação com comentários inline, notificações e registo de alterações.

**04 // DESIGN — paleta:**
- BACKGROUND: `#020807` — fundo ultra-dark teal
- SURFACE: `#0c0c0c` — cards e painéis
- ACCENT: `#35998f` — teal primário
- CTA: `#f0d474` — gold para acções principais
- TEXT: `#dcdcdc` — texto primário

**05 // STACK:**
- Next.js 14 — Framework React com App Router e Server Components
- Supabase — Backend-as-a-service: auth, base de dados, realtime
- TanStack Query — Gestão de estado servidor com cache inteligente
- TypeScript — Tipagem estática em todo o projecto
- Radix UI — Componentes acessíveis headless
- Sonner — Sistema de notificações toast

### [02] Hidden Hype

**Ficheiro:** `projeto/hidden-hype.html`
**Slug:** `hidden-hype`
**Cor de marca:** `#C9A84C` (gold)
**PREV:** `video-taskmanager.html` | **NEXT:** `hype-gest.html`

**Hero:**
- Nome: `HIDDEN HYPE`
- Categoria: `EVENTOS & TICKETING`
- URL: `hiddenhype.nodegrid.dev`

**02 // SOBRE:**
- Headline: *"Check-in manual em eventos com 500 pessoas. Resolvemos isso com um QR code."*
- Problema: Eventos com muitos participantes dependem de listas em papel ou Excel para check-in, criando filas, erros e falta de dados em tempo real para os organizadores.
- Solução: App PWA com check-in por QR code e autenticação por telemóvel — sem instalação, funciona offline, dados em tempo real no painel de admin.
- Métricas: `STACK: 5 tecnologias` | `TIPO: PWA` | `AUTH: Phone OTP` | `OFFLINE: Sim`

**03 // FUNCIONALIDADES:**
1. **Check-in por QR Code** — Scanner instantâneo via câmera do telemóvel com validação em tempo real, feedback visual e sonoro e prevenção de entradas duplas.
2. **Autenticação OTP** — Login sem password via SMS — acesso imediato para staff e participantes sem necessidade de criar conta.
3. **Painel Admin** — Dashboard com contagem em tempo real de presentes, exportação de dados por evento e controlo de capacidade com alertas.

**04 // DESIGN — paleta:**
- BACKGROUND: `#0a0a0a` — preto puro
- SURFACE: `#111111` — cards e painéis
- ACCENT: `#C9A84C` — gold primário
- CTA: `#C9A84C` — gold para acções
- TEXT: `#FFFFFF` — texto primário

**05 // STACK:**
- Next.js 15 — Framework com App Router e suporte PWA nativo
- Supabase — Auth com Phone OTP, base de dados e realtime subscriptions
- next-pwa — Progressive Web App com cache offline e installability
- html5-qrcode — Scanner de QR code via câmera do browser
- TypeScript — Tipagem estática em todo o projecto

## Notas técnicas

- **Estrutura de ficheiros:** `projeto/video-taskmanager.html`, etc. — paths relativos para assets: `../logo-nodegrid.svg`, `../css/tokens.css`, `../assets/screenshots/`
- **Browser frame mockup:**
  ```html
  <div class="browser-frame">
    <div class="browser-bar">
      <div class="browser-dots"><span class="dot red"></span><span class="dot yellow"></span><span class="dot green"></span></div>
      <div class="browser-url">videotask.nodegrid.dev</div>
    </div>
    <div class="browser-content">
      <img src="../assets/screenshots/video-task-manager.png" onerror="...">
    </div>
  </div>
  ```
- **Intersection Observer para tabs:**
  ```javascript
  const observer = new IntersectionObserver(entries => {
    entries.forEach(e => {
      if (e.isIntersecting) setActiveTab(e.target.dataset.section);
    });
  }, { threshold: 0.3 });
  sections.forEach(s => observer.observe(s));
  ```
- **Hero gradient com cor de marca:**
  ```css
  .hero-project { background: radial-gradient(ellipse 60% 50% at 30% 0%, rgba(53,153,143,.15) 0%, transparent 70%), var(--bg-base); }
  ```
  Substituir `53,153,143` pela cor RGB do projecto.
- **Color block:**
  ```html
  <div class="color-block" style="background:#020807">
    <span class="cb-label">BACKGROUND</span>
    <span class="cb-hex">#020807</span>
    <span class="cb-use">Fundo principal da app</span>
  </div>
  ```
